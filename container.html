Контейнер

display

Определяет элемент как контейнер сетки и устанавливает новый контекст форматирования сетки для его содержимого.

Возможные значения:

  • grid – генерирует сетку на уровне блоков
  • inline-grid – генерирует сетку строчного уровня
.container { ⠀⠀⠀⠀display: grid | inline-grid; }

grid-template-columns, grid-template-rows

Определяет столбцы и строки сетки со списком значений, разделенных пробелами. Значения представляют размер дорожки, а пространство между ними представляет собой линию сетки.

Возможные значения:

  • <track-size> – может быть длиной (определяется стандартными единицами такими как px), процентом или долей свободного места в сетке (с использованием единицы fr)
  • <line-name> – произвольное название которое вы выберете
.container { ⠀⠀⠀⠀grid-template-columns: ... ...; ⠀⠀⠀⠀/* например ⠀⠀⠀⠀⠀⠀⠀⠀1fr 1fr ⠀⠀⠀⠀⠀⠀⠀⠀minmax(10px, 1fr) 3fr ⠀⠀⠀⠀⠀⠀⠀⠀repeat(5, 1fr) ⠀⠀⠀⠀⠀⠀⠀⠀50px auto 100px 1fr ⠀⠀⠀⠀*/ ⠀⠀⠀⠀grid-template-rows: ... ...; ⠀⠀⠀⠀/* например ⠀⠀⠀⠀⠀⠀⠀⠀min-content 1fr min-content ⠀⠀⠀⠀⠀⠀⠀⠀100px 1fr max-content ⠀⠀⠀⠀*/ }

Линии сетки автоматически получают положительные числа из этих назначений (-1 является альтернативой для самой последней строки).

Но вы можете явно назвать строки. Обратите внимание на синтаксис скобок для именования строк:

.container { ⠀⠀⠀⠀grid-template-columns: [first] 40px [line2] 50px [line3] auto [col4-start] 50px [five] 40px [end]; ⠀⠀⠀⠀grid-template-rows: [row1-start] 25% [row1-end] 100px [third-line] auto [last-line]; }

Обратите внимание, что строка может иметь более одного имени. Например, здесь вторая строка будет иметь два имени: row1-end и row2-start:

.container { ⠀⠀⠀⠀grid-template-rows: [row1-start] 25% [row1-end row2-start] 25% [row2-end]; }

Если ваше определение содержит повторяющиеся части, вы можете использовать функцию repeat() для оптимизации:

.container { ⠀⠀⠀⠀grid-template-columns: repeat(3, 20px [col-start]); }

Что эквивалентно этому:

.container { ⠀⠀⠀⠀grid-template-columns: 20px [col-start] 20px [col-start] 20px [col-start]; }

Если несколько строк имеют одно и то же имя, на них можно ссылаться по имени строки и количеству.

.item { ⠀⠀⠀⠀grid-column-start: col-start 2; }

Единица fr позволяет вам установить размер дорожки как часть свободного пространства контейнера сетки. Например, значение ниже установит для каждого элемента одну треть ширины контейнера сетки:

.container { ⠀⠀⠀⠀grid-template-columns: 1fr 1fr 1fr; }

Свободное пространство рассчитывается после любых негибких элементов. В этом примере общий объем свободного места, доступного для блоков fr, не включает 50 пикселей:

.container { ⠀⠀⠀⠀grid-template-columns: 1fr 50px 1fr 1fr; }

grid-template-areas

Определяет шаблон сетки, ссылаясь на имена областей сетки, указанные в свойстве grid-area.

Повторение имени области сетки приводит к тому, что содержимое охватывает эти ячейки. Точка означает пустую ячейку. Сам синтаксис обеспечивает визуализацию структуры сетки.

Возможные значения:

  • <grid-area-name> – имя области сетки, указанное с помощью grid-area
  • . – пустая ячейка сетки
  • none – области сетки не определены
.container { ⠀⠀⠀⠀grid-template-areas: ⠀⠀⠀⠀⠀⠀⠀⠀"<grid-area-name> | . | none | ..." ⠀⠀⠀⠀⠀⠀⠀⠀"..."; }

Пример:

.item-a { ⠀⠀⠀⠀grid-area: header } .item-b { ⠀⠀⠀⠀grid-area: main } .item-c { ⠀⠀⠀⠀grid-area: sidebar } .item-d { ⠀⠀⠀⠀grid-area: footer } .container { ⠀⠀⠀⠀display: grid; ⠀⠀⠀⠀grid-template-columns: 50px 50px 50px 50px; ⠀⠀⠀⠀grid-template-rows: auto; ⠀⠀⠀⠀grid-template-areas: ⠀⠀⠀⠀⠀⠀⠀⠀"header header header header" ⠀⠀⠀⠀⠀⠀⠀⠀"main main . sidebar" ⠀⠀⠀⠀⠀⠀⠀⠀"footer footer footer footer"; }

Этот код создаст сетку шириной четыре столбца и высотой три строки. Вся верхняя строка будет состоять из области header. Средняя строка будет состоять из двух основных областей, одной пустой ячейки и одной боковой панели. Последняя строка — footer.

Каждая строка должна иметь одинаковое количество ячеек.

Вы можете использовать любое количество соседних точек, чтобы объявить одну пустую ячейку. Пока точки не имеют пробелов между ними, они представляют собой одну ячейку.

Обратите внимание, что вы называете не линии с помощью этого свойства, а только области. Когда вы используете это свойство, строки на обоих концах областей фактически получают имена автоматически. Если имя вашей области сетки — foo, имя начальной строки строки и начального столбца области будет foo-start, а имя ее последней строки строки и последнего столбца — foo-end. Это означает, что некоторые строки могут иметь несколько имен, например, крайняя левая строка в приведенном выше примере будет иметь три имени: header-start, main-start и footer-start.

grid-template

Сокращение для установки grid-template-rows, grid-template-columns и grid-template-areas в одном объявлении.

Возможные значения:

  • none – устанавливает все три свойства в их начальные значения
  • <grid-template-rows> / <grid-template-columns> – устанавливает для grid-template-columns и grid-template-rows указанные значения соответственно и устанавливает для grid-template-areas значение none
.container { ⠀⠀⠀⠀grid-template: none | <grid-template-rows> / <grid-template-columns>; }

Это свойство также принимает более сложный, но довольно удобный синтаксис для указания всех трех свойств. Например:

.container { ⠀⠀⠀⠀grid-template: ⠀⠀⠀⠀⠀⠀⠀⠀[row1-start] "header header header" 25px [row1-end] ⠀⠀⠀⠀⠀⠀⠀⠀[row2-start] "footer footer footer" 25px [row2-end] ⠀⠀⠀⠀⠀⠀⠀⠀/ auto 50px auto; }

Это эквивалентно этому:

.container { ⠀⠀⠀⠀grid-template-rows: [row1-start] 25px [row1-end row2-start] 25px [row2-end]; ⠀⠀⠀⠀grid-template-columns: auto 50px auto; ⠀⠀⠀⠀grid-template-areas: ⠀⠀⠀⠀⠀⠀⠀⠀"header header header" ⠀⠀⠀⠀⠀⠀⠀⠀"footer footer footer"; }

Поскольку grid-template не сбрасывает неявные свойства сетки (grid-auto-columns, grid-auto-rows и grid-auto-flow), что, вероятно, и требуется в большинстве случаев, рекомендуется использовать свойство сетки вместо шаблона сетки.

column-gap, row-gap, grid-column-gap, grid-row-gap

Задает размер отступов между колонками и строками.

Возможные значения:

  • <line-size> – значение длины (определяется стандартными единицами такими как px)
.container { ⠀⠀⠀⠀/* стандартный синтаксис */ ⠀⠀⠀⠀column-gap: <line-size>; ⠀⠀⠀⠀row-gap: <line-size>; ⠀⠀⠀⠀/* старый синтаксис */ ⠀⠀⠀⠀grid-column-gap: <line-size>; ⠀⠀⠀⠀grid-row-gap: <line-size>; }

Пример:

.container { ⠀⠀⠀⠀grid-template-columns: 100px 50px 100px; ⠀⠀⠀⠀grid-template-rows: 80px auto 80px; ⠀⠀⠀⠀column-gap: 10px; ⠀⠀⠀⠀row-gap: 15px; }

Отступы создаются только между столбцами/строками и не создаются на внешних краях.

gap, grid-gap

Сокращение для row-gap и column-gap

Возможные значения:

  • <grid-row-gap> <grid-column-gap> – значение длины (определяется стандартными единицами такими как px)
.container { ⠀⠀⠀⠀/* стандартный синтаксис */ ⠀⠀⠀⠀gap: <grid-row-gap> <grid-column-gap>; ⠀⠀⠀⠀/* старый синтаксис */ ⠀⠀⠀⠀grid-gap: <grid-row-gap> <grid-column-gap>; }

Пример:

.container { ⠀⠀⠀⠀grid-template-columns: 100px 50px 100px; ⠀⠀⠀⠀grid-template-rows: 80px auto 80px; ⠀⠀⠀⠀gap: 15px 10px; }

Если row-gap не указан, ему задается такое же значение, что и column-gap.

justify-items

Выравнивает элементы сетки вдоль строчной оси (строка) (в отличии от свойства align-items, которое выравнивает по блочной оси (столбец)). Это значение применяется ко всем элементам сетки внутри контейнера.

Возможные значения:

  • start – выравнивает элементы так, чтобы они были на одном уровне с начальным краем их ячейки
  • end – выравнивает элементы так, чтобы они были на одном уровне с конечным краем их ячейки
  • center – выравнивает элементы по центру их ячейки
  • stretch – заполняет всю ширину ячейки (по умолчанию)
.container { ⠀⠀⠀⠀justify-items: start | end | center | stretch; }

Примеры:

.container { ⠀⠀⠀⠀justify-items: start; }
.container { ⠀⠀⠀⠀justify-items: end; }
.container { ⠀⠀⠀⠀justify-items: center; }
.container { ⠀⠀⠀⠀justify-items: stretch; }

align-items

Выравнивает элементы сетки по блочной оси (столбец) (в отличии от свойства justify-items, которое выравнивает по строчной оси (строка)). Это значение применяется ко всем элементам сетки внутри контейнера.

Возможные значения:

  • start – выравнивает элементы так, чтобы они были на одном уровне с начальным краем их ячейки
  • end – выравнивает элементы так, чтобы они были на одном уровне с конечным краем их ячейки
  • center – выравнивает элементы по центру их ячейки
  • stretch – заполняет всю высоту ячейки (по умолчанию)
  • baseline – выравнивание элементов по базовой линии текста.
.container { ⠀⠀⠀⠀align-items: start | end | center | stretch; }

Примеры:

.container { ⠀⠀⠀⠀align-items: start; }
.container { ⠀⠀⠀⠀align-items: end; }
.container { ⠀⠀⠀⠀align-items: center; }
.container { ⠀⠀⠀⠀align-items: stretch; }

Эффект этого свойства также можно установить для отдельных элементов сетки с помощью свойства align-self.

place-items

place-items задает свойства align-items и justify-items в одном объявлении.

Возможные значения:

  • <align-items> / <justify-items> – первое значение устанавливает align-items, второе значение justify-items. Если второе значение опущено, первое значение присваивается обоим свойствам.

Это может быть очень полезно для сверхбыстрого центрирования по обеим из осей:

.center { ⠀⠀⠀⠀display: grid; ⠀⠀⠀⠀place-items: center; }

justify-content

Иногда общий размер вашей сетки может быть меньше размера ее контейнера. Это может произойти, если все элементы сетки имеют негибкие единицы измерения, такие как px. В этом случае вы можете установить выравнивание сетки внутри контейнера. Это свойство выравнивает сетку по строчной оси (строка) (в отличии свойства от align-content, которое выравнивает сетку по блочной оси (столбец)).

Возможные значения:

  • start – выравнивает сетку так, чтобы она была на одном уровне с начальным краем контейнера
  • end – выравнивает сетку так, чтобы она была на одном уровне с конечным краем контейнера
  • center – выравнивает сетку по центру контейнера
  • stretch – изменяет размер элементов сетки, чтобы сетка могла заполнить всю ширину контейнера
  • space-around – помещает четное количество пространства между каждым элементом сетки с отступами половинного размера на дальних концах
  • space-between – размещает равномерное пространство между каждым элементом сетки, без отступов на дальних концах
  • space-evenly – помещает четное расстояние между каждым элементом сетки, включая дальние концы
.container { ⠀⠀⠀⠀justify-content: start | end | center | stretch | space-around | space-between | space-evenly; }

Примеры:

.container { ⠀⠀⠀⠀justify-content: start; }
.container { ⠀⠀⠀⠀justify-content: end; }
.container { ⠀⠀⠀⠀justify-content: center; }
.container { ⠀⠀⠀⠀justify-content: stretch; }
.container { ⠀⠀⠀⠀justify-content: space-around; }
.container { ⠀⠀⠀⠀justify-content: space-between; }
.container { ⠀⠀⠀⠀justify-content: space-evenly; }

align-content

Это свойство выравнивает сетку по блочной оси (столбец) (в отличии от свойства justify-content, которое выравнивает сетку по строчной оси (строка).

Возможные значения:

  • start – выравнивает сетку так, чтобы она была на одном уровне с начальным краем контейнера
  • end – выравнивает сетку так, чтобы она была на одном уровне с конечным краем контейнера
  • center – выравнивает сетку по центру контейнера
  • stretch – изменяет размер элементов сетки, чтобы сетка могла заполнить всю высоту контейнера
  • space-around – помещает четное количество пространства между каждым элементом сетки с отступами половинного размера на дальних концах
  • space-between – размещает равномерное пространство между каждым элементом сетки, без отступом на дальних концах
  • space-evenly – помещает четное расстояние между каждым элементом сетки, включая дальние концы
.container { ⠀⠀⠀⠀align-content: start | end | center | stretch | space-around | space-between | space-evenly; }

Примеры:

.container { ⠀⠀⠀⠀align-content: start; }
.container { ⠀⠀⠀⠀align-content: end; }
.container { ⠀⠀⠀⠀align-content: center; }
.container { ⠀⠀⠀⠀align-content: stretch; }
.container { ⠀⠀⠀⠀align-content: space-around; }
.container { ⠀⠀⠀⠀align-content: space-between; }
.container { ⠀⠀⠀⠀align-content: space-evenly; }

place-content

place-content задает свойства align-content и justify-content в одном объявлении.

Возможные значения:

  • <align-content> / <justify-content> – первое значение устанавливает align-content, второе значение — justify-content. Если второе значение опущено, первое значение присваивается обоим свойствам.

Все основные браузеры, кроме Edge, поддерживают сокращенное свойство place-content.

grid-auto-columns, grid-auto-rows

Определяет размер любых автоматически сгенерированных дорожек сетки (так называемых неявных дорожек сетки). Неявные дорожки создаются, когда элементов сетки больше, чем ячеек в сетке, или когда элемент сетки размещается за пределами явной сетки.

Возможные значения:

  • <track-size> – может быть длиной (определяется стандартными единицами такими как px), процентом или долей свободного места в сетке (с использованием единицы fr)
.container { ⠀⠀⠀⠀grid-auto-columns: <track-size> ...; ⠀⠀⠀⠀grid-auto-rows: <track-size> ...; }

Чтобы проиллюстрировать, как создаются неявные дорожки сетки, посмотрите на пример ниже:

.container { ⠀⠀⠀⠀grid-template-columns: 60px 60px; ⠀⠀⠀⠀grid-template-rows: 90px 90px; }

Это создает сетку 2 x 2.

Но теперь представьте, что вы используете grid-column и grid-row для позиционирования элементов сетки следующим образом:

.item-a { ⠀⠀⠀⠀grid-column: 1 / 2; ⠀⠀⠀⠀grid-row: 2 / 3; } .item-b { ⠀⠀⠀⠀grid-column: 5 / 6; ⠀⠀⠀⠀grid-row: 2 / 3; }

Мы задали, что .item-b начинаться со столбца 5 и заканчиваться столбцом 6, но мы никогда не определяли столбец 5 или 6. Поскольку мы ссылались на строки, которые не существуют, неявные дорожки с шириной 0 создаются для заполнения в отступах. Мы можем использовать grid-auto-columns и grid-auto-rows, чтобы указать ширину этих неявных дорожек:

.container { ⠀⠀⠀⠀grid-auto-columns: 60px; }

grid-auto-flow

Если у вас есть элементы сетки, которые вы явно не размещаете в сетке, срабатывает алгоритм автоматического размещения элементов. Это свойство задает, как именно будет работать алгоритма автоматического размещения.

Возможные значения:

  • row – указывает алгоритму автоматического размещения заполнять каждую строку по очереди, добавляя новые строки по мере необходимости (по умолчанию)
  • column – указывает алгоритму автоматического размещения заполнять каждый столбец по очереди, добавляя новые столбцы по мере необходимости
  • dense – указывает алгоритму автоматического размещения попытаться заполнить дыры в сетке, если появятся более мелкие элементы
.container { ⠀⠀⠀⠀grid-auto-flow: row | column | row dense | column dense; }

Обратите внимание, что dense изменяет только визуальный порядок ваших элементов и может привести к тому, что они будут отображаться не по порядку, что может помешать в будущем.

Примеры:

Рассмотрим этот HTML код:

<section class="container"> ⠀⠀⠀⠀<div class="item-a">item-a</div> ⠀⠀⠀⠀<div class="item-b">item-b</div> ⠀⠀⠀⠀<div class="item-c">item-c</div> ⠀⠀⠀⠀<div class="item-d">item-d</div> ⠀⠀⠀⠀<div class="item-e">item-e</div> </section>

Мы определяем сетку с пятью столбцами и двумя строками и устанавливаем для grid-auto-flow значение row (что также используется по умолчанию).:

.container { ⠀⠀⠀⠀display: grid; ⠀⠀⠀⠀grid-template-columns: 60px 60px 60px 60px 60px; ⠀⠀⠀⠀grid-template-rows: 30px 30px; ⠀⠀⠀⠀grid-auto-flow: row; }

При размещении элементов на сетке мы указываем места только для двух из них:

.item-a { ⠀⠀⠀⠀grid-column: 1; ⠀⠀⠀⠀grid-row: 1 / 3; } .item-e { ⠀⠀⠀⠀grid-column: 5; ⠀⠀⠀⠀grid-row: 1 / 3; }

Поскольку мы установили для grid-auto-flow значение row, наша сетка будет выглядеть так. Обратите внимание, как три элемента, которые мы не разместили (элемент-b, элемент-c и элемент-d), перемещаются по доступным строкам:

item-a
item-b
item-c
item-d
item-e

Если мы вместо этого установим grid-auto-flow в column, item-b, item-c и item-d будут распределяться вниз по столбцам.:

.container { ⠀⠀⠀⠀display: grid; ⠀⠀⠀⠀grid-template-columns: 60px 60px 60px 60px 60px; ⠀⠀⠀⠀grid-template-rows: 30px 30px; ⠀⠀⠀⠀grid-auto-flow: column; }
item-a
item-b
item-c
item-d
item-e

grid

Сокращение для установки всех следующих свойств в одном объявлении: grid-template-rows, grid-template-columns, grid-template-areas, grid-auto-rows, grid-auto-columns и grid-auto-flow. (Примечание: вы можете указать явные или неявные свойства сетки только в одном объявлении сетки).

Возможные значения:

  • none – первое значение устанавливает align-content, второе значение — justify-content. Если второе значение опущено, первое значение присваивается обоим свойствам.
  • <grid-template> – работает так же, как сокращение grid-template.
  • <grid-template-rows> / [ auto-flow && dense? ] <grid-auto-columns>? – устанавливает grid-template-rows в указанное значение. Если ключевое слово auto-flow находится справа от слеша, оно устанавливает grid-auto-flow в столбец. Если дополнительно указано ключевое слово dense, алгоритм авторазмещения использует dense алгоритм расположения. Если параметр grid-auto-columns опущен, для него устанавливается значение auto.
  • [ auto-flow && dense? ] <grid-auto-rows>? / <grid-template-columns> – устанавливает grid-template-columns в указанное значение. Если ключевое слово auto-flow находится слева от слеша, оно устанавливает для grid-auto-flow значение row. Если дополнительно указано ключевое слово dense, алгоритм авторазмещения использует dense алгоритм расположения. Если параметр grid-auto-rows опущен, для него устанавливается значение auto.

Примеры:

Следующие два блока кода эквивалентны:

.container { ⠀⠀⠀⠀grid: 100px 300px / 3fr 1fr; }
.container { ⠀⠀⠀⠀grid-template-rows: 100px 300px; ⠀⠀⠀⠀grid-template-columns: 3fr 1fr; }

Следующие два блока кода эквивалентны:

.container { ⠀⠀⠀⠀grid: auto-flow / 200px 1fr; }
.container { ⠀⠀⠀⠀grid-auto-flow: row; ⠀⠀⠀⠀grid-template-columns: 200px 1fr; }

Следующие два блока кода эквивалентны:

.container { ⠀⠀⠀⠀grid: auto-flow dense 100px / 1fr 2fr; }
.container { ⠀⠀⠀⠀grid-auto-flow: row dense; ⠀⠀⠀⠀grid-auto-rows: 100px; ⠀⠀⠀⠀grid-template-columns: 1fr 2fr; }

Следующие два блока кода также эквивалентны:

.container { ⠀⠀⠀⠀grid: 100px 300px / auto-flow 200px; }
.container { ⠀⠀⠀⠀grid-template-rows: 100px 300px; ⠀⠀⠀⠀grid-auto-flow: column; ⠀⠀⠀⠀grid-auto-columns: 200px; }

Он также принимает более сложный, но довольно удобный синтаксис для установки всего сразу. Вы указываете grid-template-areas, grid-template-rows и grid-template-columns, а всем остальным подсвойствам присваиваются их начальные значения. Что вы делаете, так это указываете имена линий и размеры дорожек вместе с соответствующими областями сетки. Проще всего это описать на примере:

.container { ⠀⠀⠀⠀grid: [row1-start] "header header header" 1fr [row1-end] ⠀⠀⠀⠀⠀⠀⠀⠀[row2-start] "footer footer footer" 25px [row2-end] ⠀⠀⠀⠀⠀⠀⠀⠀/ auto 50px auto; }

Это эквивалентно этому:

.container { ⠀⠀⠀⠀grid-template-areas: ⠀⠀⠀⠀⠀⠀⠀⠀"header header header" ⠀⠀⠀⠀⠀⠀⠀⠀"footer footer footer"; ⠀⠀⠀⠀grid-template-rows: [row1-start] 1fr [row1-end row2-start] 25px [row2-end]; ⠀⠀⠀⠀grid-template-columns: auto 50px auto; }
